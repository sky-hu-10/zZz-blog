模块化

AMD与CommonJS的区别

AMD是明确面向浏览器环境的。具有自动处理依赖，异步加载模块，可以在同一个文件定义多个模块

CommonJS是基于文件系统的模块。规定一个文件只能定义一个模块。CommonJS提供一个module变量，该变量具备exports属性通过exports暴露公共接口，通过require引入模块。CommonJS是node默认的模块格式。

Es6模块化结合AMD和CommonJS的特点。基于文件系统，一个文件是一个模块，支持异步模块加载，通过关键字export导出标识符，在其他模块使用import导入标识符。模块支持默认导出export default， export和import都可以过痛关键字as使用别名导出导入标识符

事件循环

宏任务：创建主文档对象，解析html，执行主线javascript 代码，更改当前url以及各做事件。

微任务： promise回调函数，dom发生变化，微任务需要尽可能快并且通过异步方式执行，同时不能产生新的微任务。

事件循环：事件循环至少有两个队列，一个宏任务和微任务。一次处理一个任务，一个任务开始后知道结束不会被其他任务中断。

事件循环开始时先检查是否有宏任务然后再检查微任务所有微任务处理完之后才走向下一步检查是否重新渲染UI。单次循环迭代中，最多处理一个宏任务，而当前队列中的所有微任务将全部处理。
