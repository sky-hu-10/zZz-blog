React 组件通信

  1. props 单向数据流自上而下

    react 组件间通信最常见的就是通过porps在父子组件通信。

    父组件通过jsx的属性对象的方式传递给子组件，子组件通过props参数（函数组件）或者this.props（class组件）的对象接收。

    React和Vue 一样都希望保持props的只读性，不要在组件内更改props。

    既然不推荐组件内修改props,我们可以通过回调函数的形式通知父组件修改对应state。这也是子组件向父组件通信的一个方式。

    props 支持展开运算符... 来将整个props对象传递，其原理是在jsx编译成React.createElement()函数后存在props对象时会通过_extends()函数将所有props扁平化。

  2. context 方法支持向多个子孙组件传递

    使用React.createContext(defalutValue)创建一个Context对象，defalutValue为创建的Context对象默认值，只有当消费组件Consumer没有匹配到 Provider时 defalutValue才会生效 。

    Context对象提供Provider,Consumer两个组件 ,Provider可以视为分发组件其接受一个value属性，传递给消费组件Consumer。

    在类组件中通过将挂载的contextType属性重新复制为React.createContext(defalutValue)创建的context对象，这样在类组件任何生命周期都可以通过this.context来获取context对象上的value属性

    函数组件可以作为Consumer 子组件利用render props 的形式接收Provider分发的value值

    动态context可以在Provider 的value属性中添加对应的修改函数，消费组件接收并触发函数。

    需要注意Provider会根据value来决定何时重新渲染并且每次渲染都会重新渲染其下所有的Consumer组件。所以防止意外渲染可以将value的状态提升到父节点的state中。

    context 使用

  3. 构建一个全局的观察者Eventbus

    熟悉vue的同学都应该有了解vue 中有一个事件总线的概念即Eventbus。它让所有组件公用一个事件中心，提供接收，监听，移除三个方法，从而实现跨组件的通行。其内部原理是发布/订阅的设计模式。因此我们完全可以在React的项目中自己创建一个这样的事件中心。

  4. 使用refs来获取组件实例实现组件通信

    React提供了Refs来允许我们访问dom节点或者react 组件实例。因此我们可以在父组件中通过refs来调用子组件方法来进行通信

    更多refs使用可以查看我codepen 中的代码
