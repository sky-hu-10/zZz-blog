Vue项目总结

Dec 26, 2019 | Vue

Vue

前言

经历了三个月的996所谓冲刺时间，总算能有空闲时间总结一下这个项目的收获了。这次做的是一个金融类管理后台，前端架构基于Vue 2.x + element。由于是多人协同开发，项目工期赶，所有代码复用就尤其的重要。本文将总我自己在Vue项目中提高代码复用的几种方法。

组件

本次作为项目总结,仅我把自己在项目中对组件的运用记录下来。

在Vue中组件是可复用的Vue实列，且带有一个名字。因为组件是可复用的Vue实列，所有拥有与new Vue接受同样的选项，但与之不同但为 data 必须为一个函数

对于这一点我理解为:因为vue中组件都是需要独立的内部状态，由于js语言的特殊性，通过将数据作为返回对象的函数来避免每一次实例化都引用了同一个对象，从而改变其中一个时另一个也被改变

组件设计

  1. 基础库的建设
    在大型后台应用中，基础库的抽取能够帮我们节省大量开的开发时间。虽然基础库的抽取是很头疼的事情，但是现今市面上已经有很多的ui库供我们选择，也提供了定制化的方法。所有这一次我们公司的项目是在element基础上与设计定了一套自己的规范。
  2. 业务组件的建设
    业务组件我一般根据是否需要共享数据来区分，因为在实际开发中确实存在组件需要组件外的数据，例如一些依赖用户体系的组件。

组件的注册

  1. 全局注册 对于一些基础的，使用场景多的组件，可能需要全局注册这样可以在需要的地方直接使用<组件名></组件名>
  2. 局部注册 虽然全局注册组件能够极大的提高代码的复用率，但是同样也带来了性能的消耗(由于挂在全局所有进入页面是就会加载)。在对组件使用率没那么高的情况下局部注册则更适合。
    局部注册最大的特点就是能哪些业务逻辑相对独立的组件独立维护，耦合度更低这样业务逻辑也更清晰。

组件插槽

在一些具体的业务中可能存在一些场景，由固定的布局框架和动态的内容构成，比如弹窗。还有时候产品大佬一句话，需求改变需要将之前固定的组件改变位置。这个时候Vue提供的用来做内容分发的组件插槽就很适合我们。

混入（Mixins）

官方对于混入的定义为：

混入 (mixin) 提供了一种非常灵活的方式，来分发 Vue 组件中的可复用功能。一个混入对象可以包含任意组件选项。当组件使用混入对象时，所有混入对象的选项将被“混合”进入该组件本身的选项。

可以简单的理解为混入是对可以复用的代码块的使用。在管理系统中经常会有一些权限不同，但数据结构和展示形式相同的组件这种情况下混入的方式就很适合使用。我们可以将数据处理函数和数据定义抽离出来构建一个mixins/myMixin.js

使用混入时需要注意的是,混入与组件合并时对于存在同名钩子函数和普通对象的处理机制有所差别。对于data,methods,components等对象存在同名时会覆盖混入的而优先使用组件中存在的，然而对于钩子函数却会保留两边的，并先执行混入的钩子函数。
提到mixins那么extends也不得不提，毕竟在官方给二者都定义为继承作用。但与之不同的是extends接受的是对象或者函数，当同时存在mixins与extends且钩子函数同名时extends会优先触发。
当我思考extends与mixins使用场景区分时，查询了许多资料好像也都没有很好的诠释他们使用场景都细分。在官方解释中

mixins 选项接收一个混入对象的数组。这些混入对象可以像正常的实例对象一样包含实例选项，这些选项将会被合并到最终的选项中，使用的是和 Vue.extend() 一样的选项合并逻辑。也就是说，如果你的混入包含一个 created 钩子，而创建组件本身也有一个，那么两个函数都会被调用。
extends 允许声明扩展另一个组件(可以是一个简单的选项对象或构造函数)，而无需使用 Vue.extend。这主要是为了便于扩展单文件组件。
从可接受的参数结构来看，确实extends更适合单文件组件的扩展。而mixins可能存在多个mixins混入导致组件功能过于混淆的情况。

自定义指令

在Vue中指令除了内置指令外也提供了自定义指令的方法。同样的注册指令时也有挂载到全局Vue实例上的全局指令，也可以在组件中通过directives对象来注册局部组件

对于其中钩子函数以及参数我觉得官方文档上解释的更加清晰

需要补充的是在实际的业务开发中我们可能会需要获取当前绑定自定义指令的组件实例，这是时候我们可以用到vnode.context,它指向的就是对应组件的实例。
同时在开发中需注意当指令绑定值存在需要动态改变时，如果我们只使用了bind一种钩子函数来执行对应dom操作那么在组件内对应值动态改变时绑定值并不会动态改变，需要我们用update钩子函数重新进行dom操作。Vue官方为我们提供了同时应对bind和update的简写方式。

过滤器

在应对需要输出文本格式或者处理文本内容时，过滤器更适合使用。同样过滤器也是分全局注册和组件内注册

过滤器其实就是一个js函数，所以它也能接受多个参数。并且表达式的值会永远为第一个参数。Vue允许我们串联使用过滤器，后置位过滤器函数会将前置位处理之后的结果作为第一个参。

总结

在开发大型的项目时，提炼loading，persimmon这样的交互出来作为全局组件或者指令，将那些全局都有可能用到对应数据转为相应文本定义为全局的过滤器。这些操作都能大量的节省开发时间，也提高了代码的可读性。同时需要提到的是Vue的官方文档写得确实很好，仔细阅读文档能解决大部分问题。所以在以后的学习中，读文档的能力是需要大量练习和提高的。
