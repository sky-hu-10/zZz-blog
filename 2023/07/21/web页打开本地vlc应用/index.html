web页打开本地vlc应用

Jul 21, 2023 | 笔记

背景介绍

​ 提高对接入方视频流兼容性，提升审核质量，对视频播放有兜底策略。因此希望能打通系统与第三方播放器的数据连通，降低培训成本。

解决方案

​ 目前使用的方案为手动复制URL地址用的vlc播放，所以第一时间想到的是看有没有浏览器插件播放。一顿搜索找到一Open with VLC的谷歌插件，可是由于当视频流为HEVC编码时我们在低版本上采用了wasm软解导致VLC插件检测不到播放流，而且Chrome插件限制了浏览器因此这个方案只能放弃。但是插件的方案给了我们直接利用浏览器直接调起本地VLC播放的思路。

唤起本地应用

​ 唤起本地应用最主要知道对应的唤醒协议，在Windows中我们可以通过对应注册表查找，macOS通过对应URL Schemes下我们可以唤醒本地应用，打开finder => 应用程序 => 右击应用并选择显示包内容 => 选择Contents文件夹下的Info.plist文件可以查询对应协议。我们查看VLC官方包发现并没有注册对应的唤起协议，但是分析Open with VLC开源代码找到对应了唤起协议，并且作者也开源对应的插件包vlc-protocol

优化交互

​ 唤起本地应用的交互大家都应该体验过，当我们本地并没有注册对应协议时就会操作后没有任何响应。我们需要检测本地是否有对应协议，直接上代码。

总结

​ 面对那些陌生的需求和功能时我们应该分析需求关键点，用好搜索引擎。因为既然是正常需求那么必然有对应的解决方案，这个打开vlc应用的需求我们可以看到并没有需要很多代码。善用搜索引擎也是我们代码能力的一部分。
